Guia de Acesso e Documentação do Servidor Nutec

Este documento serve como um guia central para a nova infraestrutura de TI do setor Nutec. O objetivo deste projeto é centralizar nossas ferramentas, automatizar processos e criar uma base sólida para futuros serviços, conforme detalhado no relatório de projetos de TI.

A Arquitetura da Solução

Toda a nossa nova infraestrutura está consolidada em um único servidor físico, gerenciado pela plataforma de virtualização Proxmox. Isso nos dá flexibilidade e segurança, separando cada serviço em seu próprio ambiente isolado.

A estrutura funciona em camadas:

  • Servidor Físico: A máquina que hospeda tudo.
  • Proxmox VE: O sistema principal (hipervisor) que nos permite criar e gerenciar Máquinas Virtuais (VMs).
  • Máquinas Virtuais (VMs): Temos duas máquinas virtuais isoladas onde rodamos nossos sistemas: a VM Gateway e a VM Aplicações.
  • Docker: Dentro de ambas as VMs, usamos o Docker para rodar cada serviço em seu próprio contêiner, garantindo portabilidade e facilidade na gestão.

Configuração das Máquinas Virtuais (VMs)

Para otimizar o desempenho do nosso servidor (i3-8100, 16GB RAM, SSD 240GB), cada Máquina Virtual foi dimensionada para atender às necessidades de suas aplicações. Abaixo, detalhamos a alocação de recursos:

Recursos do Servidor Físico:

  • Processador: Intel Core i3-8100 (4 Cores)
  • Memória RAM: 16 GB DDR4
  • Armazenamento: SSD 240 GB

Detalhes das VMs:

VM e Aplicações Sistema Operacional CPU RAM Disco Finalidade
VM Gateway
(Nginx/Traefik, Cloudflare Tunnel, Docker)		 Debian 12 / Ubuntu Server 2 Cores 2 GB 30 GB (Thin) Ponto de entrada para acesso externo seguro e proxy reverso.
VM Aplicações
(GLPI, DocuSeal, Zabbix, Grafana, Docker)		 Debian 12 / Ubuntu Server 4 Cores 12 GB 150 GB (Thin) Consolida as principais aplicações de gestão e monitoramento.
TOTAL ALOCADO - 6 Cores 14 GB 180 GB -

*Thin Provisioning: O espaço em disco é reservado, mas só ocupa o espaço real no SSD conforme os dados são gravados.

Observação sobre CPU: A alocação total de CPU (6 cores) excede os 4 cores físicos do servidor. Isso é conhecido como "oversubscription" e é uma prática comum em virtualização, pois raramente todas as VMs utilizarão 100% de seus recursos de CPU simultaneamente. O Proxmox gerenciará a prioridade de forma eficiente.

Como Acessar o Proxmox

O Proxmox é o painel de controle de todo o servidor. A partir dele, podemos gerenciar as VMs, verificar o consumo de recursos e administrar toda a infraestrutura. O acesso é restrito à rede interna da WM Saúde.

  1. Pré-requisito: Você deve estar conectado à rede local da WM Saúde (via cabo ou Wi-Fi).
  2. Acesse o endereço: Abra seu navegador e acesse a seguinte URL. Lembre-se de usar https:// e a porta :8006.
    https://10.0.0.10:8006
  3. Aviso de Segurança: O navegador exibirá um alerta de "Conexão não segura". Isso é normal. Clique em "Avançado" e depois em "Continuar para 10.0.0.10 (não seguro)".
  4. Tela de Login: Use as seguintes credenciais:
    • Nome de usuário: `root`
    • Senha: (A senha que você definiu na instalação)
    • Realm: `Proxmox VE authentication server`

Nota Importante: O Proxmox é o "coração" da nossa infraestrutura. Tenha cuidado ao realizar qualquer alteração. Este acesso deve ser utilizado para monitoramento e gerenciamento planejado das VMs.
